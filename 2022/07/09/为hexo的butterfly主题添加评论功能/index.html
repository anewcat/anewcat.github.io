Hexo搭建的博客中引入Gitalk进行评论

Created2022-07-09|Updated2022-07-09|Hexo搭建博客
|Post View:

博客评论功能的实现

本文记录了butterfly主题中使用Hexo的集成Gitalk评论系统的步骤

修改配置

首先在主题配置文件_config.yml中修改一些配置

集成

在github上注册新应用

1、打开github网站登陆后，点击右上角的用户图标，选择 Settings。
2、 在Settings页面选择 Developer settings 选项。
3、在Developer settings选择 OAuth Apps, 然后会在页面右边有一个 New OAuth App 按钮，点击这个按钮就进入到新建 OAuth application 页面。

其中各个参数说明如下

Application name： # 应用名称，可以随意
Homepage URL： # 网站URL，自己的网站地址（这里最好设置为 用户名.github.io 注意要精确到https）
Application description # 描述，随意
Authorization callback URL：#这里填写的是github登录后要返回的地址，填自己要使用Gitalk的博客地址，不可乱填，并且如果注册了域名这里一定要写注册的域名并且要精确到https，否则可能会不生效

填写完成可以得到Client ID和Client Secret

申请评论存储仓库

github上申请一个仓库做评论存储，记住仓库名，到时需要填入hexo的_config.yml配置中

修改完以上配置后分别执行 hexo clean hexo g hexo d 即可同步更新。
